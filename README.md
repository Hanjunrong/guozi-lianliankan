一款经典的连连看网页游戏，主题为郭子老师表情包。纯 HTML/CSS/JS 实现，无需构建，开箱即玩。

## 项目亮点

### 1. 经典 BFS 路径搜索算法
- 支持 **0拐点、1拐点、2拐点** 三种连接方式
- 路径可延伸到棋盘边界外（边框外连线）
- 自动寻找最优路径（拐点最少、步数最短）
- 智能剪枝：优先扩展更优路径，大幅减少搜索空间

### 2. 流畅的交互体验
- **悬停预判连线**：鼠标移动到可消除的方块时，实时预览连接路径
- **消除动画**：成功匹配时绘制蓝色连线动画
- **错误提示**：匹配失败时绘制红色闪烁路径
- **计时器系统**：5秒无操作自动高亮提示可消除的对

### 3. 丰富的游戏功能
- **一键消除**：自动找到所有可消除的对并逐一消除
- **自动重排**：无路可走时自动检测并询问是否重排
- **音效系统**：成功/失败音效反馈（可开关）
- **坐标轴辅助**：行列坐标标注，方便定位

### 4. 工程化质量
- **100% 测试覆盖**：30 个单元测试，覆盖所有核心逻辑
- **代码分离**：游戏逻辑(game.js)、UI逻辑(ui.js)、样式(style.css) 分离
- **双环境运行**：同时支持 Node.js 测试和浏览器运行
- **ES5 兼容**：兼容所有现代浏览器

## 游戏规则

1. 点击一个方块选中它
2. 再点击另一个相同图案的方块
3. 如果两者之间存在**不超过2个拐点**的路径，方块即被消除
4. 消除所有方块即可通关

## 技术实现

### 路径搜索算法
```
findPath(r1, c1, r2, c2) 使用 BFS：
- 网格扩展：允许路径延伸到边界外（坐标 -1 和 ROWS/COLS）
- 状态表示：(row, col, direction, turns, steps, path)
- 剪枝策略：
  - 拐点数 > 2 直接跳过
  - 同一位置同一方向，保留更优（更少拐点/步数）的状态
  - 找到目标后，优于当前最优解才更新
```

### 文件结构
```
├── index.html      # 游戏页面
├── game.js        # 游戏核心逻辑（可测试）
├── ui.js          # UI 交互逻辑
├── style.css      # 样式
├── server.js      # 静态文件服务器 (port 8888)
├── run.sh         # 启动脚本
├── demo.mp4       # 演示视频
├── game.test.js   # 单元测试 (30 tests)
└── assets/        # 静态资源（图片、音效）
```

## 快速开始

```bash
# 方式一：运行启动脚本（自动先跑测试）
./run.sh

# 方式二：手动启动
node game.test.js   # 先跑测试
node server.js      # 启动服务器
# 访问 http://localhost:8888
```
